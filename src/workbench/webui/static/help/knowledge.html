Knowledge Base

Overview

The Knowledge Base agent lets you create collections of documents, upload files, paste text, and query your content using semantic search powered by ChromaDB vector embeddings. Think of it as your personal RAG (Retrieval-Augmented Generation) system.

Collections

Collections are named containers for documents. Each collection has independent chunking settings. Click a collection in the sidebar to select it — all subsequent uploads and queries target that collection.

Creating a Collection

Click "New Knowledge Base" and fill in:

  • Name — descriptive label for this collection
  • Description — optional notes about the collection's purpose
  • Chunk Size — size of text chunks for embedding (tokens)
  • Chunk Overlap — overlap between consecutive chunks (tokens)

Click "Create" to save or "Cancel" to abort.

Collection List

All your collections appear on the left. Click a collection to select it. The main area updates to show that collection's documents and query interface.

File Upload

Drag and drop files onto the upload zone or click to browse. Supported formats include .txt, .pdf, .md, .html, and more. Uploaded files are chunked according to the collection's settings and embedded into the vector database.

Paste Text

Expand the "Paste Text" section to manually enter content. Provide a filename (for identification in the document list) and paste your text content. Click "Ingest Text" to chunk and embed the content.

Document Management

Documents in the selected collection are listed with their names and sizes. Click the delete button (trash icon) next to any document to remove it from the collection and vector store.

Querying

Type your question in the query input and click "Send" (or press Enter). The agent retrieves the most relevant chunks from the vector store and feeds them to the LLM for a contextual answer. Results show both the answer and the source document excerpts.

Tip: Adjust chunk size based on your content type — smaller chunks (256-512) for Q&A, larger chunks (1024+) for document summarization.
