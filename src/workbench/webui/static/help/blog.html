Blog / Publishing Hub

Overview

The Blog/Publishing Hub lets you create and manage documents in Markdown, HTML, or PDF format. Every document is automatically versioned via git (transparent to the user) with full history inspection. Published documents appear on your public blog page at /blog/{username}.

New Document

Click "New Document" to create a blank post. Fill in:

  • Title — the document title (auto-generates a URL slug)
  • Format — Markdown, HTML, or PDF. Markdown is rendered to HTML on the public page
  • Content — the main document body (textarea for text formats)
  • Comment — an optional git commit message describing the change (max 2048 chars)
  • Published — toggle to make the document visible on your public blog

File Upload

Drag and drop a file onto the upload area, or click to browse. Supported formats: .md, .html, .pdf. Uploaded files populate the content automatically.

Save & Delete

  • Save — commits the current state to git and updates the database. Shows loading state during save.
  • Delete — permanently removes the document and its git history. Confirmation required.
  • Cancel — discards unsaved changes and returns to the post list

Post List

Shows all your documents with title, format, and published status. Click a post entry to edit it.

Version History

Click "History" while editing a document to open the inspection overlay. Shows the git commit log with timestamps, comments, and commit hashes. Click a commit hash to view that version of the document. The overlay is read-only — your current edit is preserved underneath.

Public Blog

Published documents appear on your public page at /blog/{your-username}. PDFs show as download cards. HTML content is sanitized (scripts and event handlers removed). Markdown is rendered to styled HTML.

Tip: Use the comment field to document why you made a change — it makes the git history more useful for later inspection.
